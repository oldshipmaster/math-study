# 数学互动学习网站设计说明

日期：2026-07-11

## 目标

为 7–12 岁儿童制作一个中文数学学习网站。每个章节由浏览器原生 HTML、CSS 和 JavaScript 驱动，以动画讲解、点击探索、拖放练习和即时答题为主要学习方式。

首版完整上线两节课：

1. 数字符号：来源为 B 站视频 `BV1Cuu8zcERv` 的分 P 1。
2. 位值：来源为同一视频的分 P 2。

网站仅提炼和重新组织数学知识，不复制原视频画面、配音或逐句文案。

## 受众与体验原则

- 面向 7–12 岁，表达直观但不过度幼龄化。
- 学生通过操作理解概念，不以被动观看长动画为主。
- 答错可以重试，不扣分；反馈用于解释原因，而不是制造压力。
- 同时支持桌面鼠标、触屏和键盘操作。
- 尊重系统的“减少动态效果”设置。

## 视觉方向

采用“编辑感数学实验室”方向：

- 米白背景、墨绿正文、橙红重点色、金黄辅助色。
- 大字号衬线标题搭配清晰的无衬线正文。
- 视觉气质接近一本可互动的高品质数学科普书。
- 使用排版、色块、CSS 图形和数字卡片表达知识，不依赖装饰性插画。

## 信息架构

- 首页首先展示课程价值、当前学习进度和课程章节。
- 课程按“数与数感、四则运算、分数与小数、测量、几何”分组。
- 首版开放“数字符号”和“位值”；其他已提取的课程名称可作为即将开放的目录项。
- 课程页左侧为章节导航，主区为 16:9 动画舞台，底部提供上一步、重播和继续。
- 小屏设备将章节导航收纳为抽屉，动画舞台和操作区纵向排列。

## 通用课程模型

每节课由相互独立的场景构成。每个场景具有四种状态：

1. 进入：呈现场景目标。
2. 讲解：用短动画解释一个概念。
3. 互动：学生点击、拖放或答题。
4. 完成：给出反馈并点亮进度节点。

场景内容通过课程数据配置，通用舞台负责进度、导航、反馈和动画编排。后续新增课程时复用舞台与互动组件。

## 第一课：数字符号

学习目标：理解数量与数字符号的区别，认识同一数量可以用不同符号表达。

场景：

1. 数量先于数字：点击物体并观察数量变化。
2. 符号的诞生：从刻痕和手指数数过渡到现代数字符号。
3. 一种数量，多种写法：将 `3`、`三`、`Ⅲ` 与三个物体配对。
4. 拖动挑战：把数量卡拖入正确的数字容器。
5. 终局测验：三道动态题，完成后生成星级和知识卡。

## 第二课：位值

学习目标：理解数字的位置决定其数值，掌握个位、十位和百位的基本关系。

场景：

1. 十个一变成一个十：用可组合的单位块演示进位。
2. 数位屋：把数字卡拖入百位、十位和个位。
3. 拆解数字：把 `352` 动态展开为 `300 + 50 + 2`。
4. 位置改变，价值改变：比较 `305` 和 `350` 中数字 `5` 的价值。
5. 造数挑战：按目标数值拖放百块、十条和单位块。
6. 终局测验：三道动态题，完成后生成星级和知识卡。

## 互动与反馈

- 点击题提供明确的可点击状态和焦点状态。
- 拖放题支持鼠标、触屏和键盘替代操作。
- 错误拖放自动归位，并用短文字或局部动画解释错误。
- 每个场景可重播；返回上一场景不清除已经获得的完成状态。
- 结课页展示正确率、尝试次数和三个已掌握知识点。

## 状态与数据

- 首版不要求注册或登录。
- 当前课程、场景进度、答题记录和结课结果保存在浏览器本地。
- 刷新或下次访问时可继续学习。
- 本地数据不可用时退化为当前会话内学习，不阻塞课程操作。

## 无障碍与适配

- 主要文字与背景满足清晰对比。
- 所有互动元素具有可见焦点和中文无障碍标签。
- 不依赖颜色单独传达正确或错误。
- 动画在系统要求减少动态效果时改用淡入、瞬时位移或静态状态。
- 支持常见手机、平板和桌面宽度。

## 验收标准

- 两节课程均能从开始完整进行到结课。
- 点击、拖放、键盘替代操作和答题反馈工作正常。
- 刷新后能够恢复课程进度。
- 桌面和手机布局无关键内容遮挡或横向溢出。
- 减少动态效果模式下仍能理解所有知识变化。
- 生产构建成功，网站可由 Sites 发布并访问。

## 非首版范围

- 用户账号、云端同步、排行榜和多人功能。
- 教师后台、家长报告和付费系统。
- 自动抓取或直接播放 B 站视频内容。
- 首版两课之外的完整互动课程制作。
