# 重置课程进度功能设计

日期：2026-07-11

## 目标

允许家长按课程清空当前浏览器中保存的学习进度，也可在明确选择后一次清空全部课程，让孩子从指定课程的第一场景重新开始。

## 入口与交互

- 首页顶部在“已完成 X/5 课”状态旁显示“重置进度”按钮。
- 手机端在课程区域顶部提供同一入口，避免入口随进度状态一起隐藏。
- 点击按钮后打开课程管理面板，不预选课程、不直接清除数据。
- 面板列出全部 5 节课程及各自当前进度，每节提供“重置本课”按钮。
- 点击“重置本课”后显示该课程专属二次确认；确认后只清除该课程，其他课程保持不变。
- 面板底部保留独立警示区“重置全部课程”，与逐课操作明显分隔。
- “重置全部课程”仍需单独的二次确认，不作为默认焦点或默认动作。
- 单课重置后该课程恢复“开始学习”和 0% 进度，并提示“《课程名》进度已重置”。
- 全部重置后首页显示“已完成 0/5 课”，全部课程恢复“开始学习”，并提示“全部课程进度已重置”。

## 数据行为

- 在 `lib/progress.ts` 提供 `clearCourseProgress(courseId)`，只删除指定课程记录并保存其余记录。
- 保留 `clearProgress()`，仅供用户明确确认“重置全部课程”后调用。
- 清空后 React 内存状态同步恢复为 `{ lessons: {} }`，无需刷新页面。
- 当本地存储不可用或删除失败时，当前页面仍恢复为空进度，保证孩子可以从头开始；不向用户显示技术错误。
- 重置只影响课程学习记录，不修改课程内容、网站设置或其他浏览器数据。

## 无障碍与防误触

- 课程管理面板与确认框使用对话框语义，并提供明确标题。
- 打开后可通过键盘操作取消或确认。
- “确认重置”使用警示视觉，“取消”为默认安全选择；打开面板时焦点不落在“重置全部课程”上。
- 不使用浏览器原生 `confirm()`，保持手机端和桌面端体验一致。

## 测试与验收

- 单元测试验证 `clearCourseProgress()` 只删除指定课程、`clearProgress()` 删除全部记录，且存储失败不会抛错。
- 界面测试验证打开面板不会清空、取消保留进度、单课确认只清除目标课程、全部确认清空全部课程。
- 运行完整自动化测试和 GitHub Pages 静态构建。
- 发布后确认新入口存在，并验证重置后第一课从第一场景开始。
