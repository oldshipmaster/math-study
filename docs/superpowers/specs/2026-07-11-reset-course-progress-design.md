# 重置课程进度功能设计

日期：2026-07-11

## 目标

允许家长在测试完成后，一次清空当前浏览器中保存的全部课程进度，让孩子从第一课、第一场景重新开始学习。

## 入口与交互

- 首页顶部在“已完成 X/2 课”状态旁显示“重置进度”按钮。
- 手机端在课程区域顶部提供同一入口，避免入口随进度状态一起隐藏。
- 点击按钮后显示站内确认对话框，不直接清除数据。
- 对话框明确说明：数字符号、位值的场景进度、答题次数和结课记录都会清空，且无法撤销。
- 用户可选择“取消”或“确认重置”；只有“确认重置”执行清空。
- 重置完成后关闭对话框，首页立即显示“已完成 0/2 课”，两张课程卡恢复“开始学习”和 0% 进度，并短暂显示“课程进度已重置”。

## 数据行为

- 在 `lib/progress.ts` 提供 `clearProgress()`，统一删除 `math-lab-progress` 本地存储键。
- 清空后 React 内存状态同步恢复为 `{ lessons: {} }`，无需刷新页面。
- 当本地存储不可用或删除失败时，当前页面仍恢复为空进度，保证孩子可以从头开始；不向用户显示技术错误。
- 重置只影响课程学习记录，不修改课程内容、网站设置或其他浏览器数据。

## 无障碍与防误触

- 确认框使用对话框语义，并提供明确标题。
- 打开后可通过键盘操作取消或确认。
- “确认重置”使用警示视觉，“取消”为默认安全选择。
- 不使用浏览器原生 `confirm()`，保持手机端和桌面端体验一致。

## 测试与验收

- 单元测试验证 `clearProgress()` 删除指定键且删除失败不会抛错。
- 界面测试验证首次点击不会直接清空、取消保留进度、确认后状态归零。
- 运行完整自动化测试和 GitHub Pages 静态构建。
- 发布后确认新入口存在，并验证重置后第一课从第一场景开始。
