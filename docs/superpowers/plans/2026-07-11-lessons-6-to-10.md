# Lessons 6–10 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add five complete, accessible HTML-animation math courses for comparison, ordering, estimation, rounding, and multiples, while preserving existing progress and reset behavior.

**Architecture:** Extend the typed course catalog with five scene kinds and five courses. Keep each interaction family in a focused React component using the existing `onComplete(correct, attempts)` contract, dispatch them from `MathLab`, and use course-data-driven reset/home rendering so the catalog grows from five to ten without a storage migration.

**Tech Stack:** React 19, TypeScript 5.9, Vitest, Testing Library, CSS animations, vinext/Vite, GitHub Pages static build.

## Global Constraints

- Audience is 7–12 years old; every lesson is an original interactive adaptation, not a reproduction of video artwork or script.
- Course 10 includes multiples, common multiples, and least common multiple as required content.
- All primary operations support mouse, touch, and keyboard; drag-style tasks must have a click alternative.
- Existing progress for lessons 1–5 must remain readable; per-course reset and reset-all must cover all ten courses.
- `prefers-reduced-motion` must preserve final states while disabling nonessential movement.
- Do not add runtime dependencies or image assets.
